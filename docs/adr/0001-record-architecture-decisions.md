# 1. Record Architecture Decisions

Date: 2024-03-19

## Status

Accepted

## Context

We need to record the architectural decisions made on this project to provide context for future development and maintenance.

## Decision

We will use Architecture Decision Records, as described by Michael Nygard in this article: http://thinkrelevance.com/blog/2011/11/15/documenting-architecture-decisions

## Consequences

See Michael Nygard's article, linked above.

Consequences of implementing this ADR:

* Easy for new team members to understand historical decisions
* Helps prevent confusion around why certain choices were made
* Provides context for future changes
* Makes architectural decisions explicit and visible
